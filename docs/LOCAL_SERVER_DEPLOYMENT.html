3. PostgreSQL Setup

Install & Start

# macOS
brew install postgresql@16
brew services start postgresql@16

# Ubuntu
sudo apt update && sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql
sudo systemctl enable postgresql

Create Databases

# Replace 'prayash' with your PostgreSQL username
createdb -U prayash mindforge
createdb -U prayash mindforge_admin
createdb -U prayash mindforge_teacher
createdb -U prayash mindforge_parent

# Verify
psql -U prayash -l
macOS Homebrew PostgreSQL: Default user is your OS username with no password.

5. Build Everything

Automated (recommended)

cd /Users/prayash/Project/Mindforge
chmod +x scripts/build-and-deploy.sh
./scripts/build-and-deploy.sh

This builds all frontends and backends, starts PM2, and runs health checks.

Manual

# Install dependencies
npm install

# Build each frontend
cd apps/student/frontend && npm run build && cd ../../..
cd apps/parent/frontend  && npm run build && cd ../../..
cd apps/teacher/frontend && npm run build && cd ../../..
cd apps/admin/frontend   && npm run build && cd ../../..

# Build each backend
cd apps/student/backend && npx nest build && cd ../../..
cd apps/parent/backend  && npx nest build && cd ../../..
cd apps/teacher/backend && npx nest build && cd ../../..
cd apps/admin/backend   && npx nest build && cd ../../..

6. Start Backends with PM2

# Start all 4 backends
pm2 start ecosystem.config.js

# Check status
pm2 status

# Enable auto-start on reboot
pm2 save
pm2 startup   # Follow the printed command

Verify

curl http://localhost:3000/health   # Student
curl http://localhost:3002/health   # Parent
curl http://localhost:3003/health   # Teacher
curl http://localhost:3004/health   # Admin

7. Configure Apache HTTP Server

Important: You need Apache HTTP Server (httpd), not Apache Tomcat. Tomcat is a Java servlet container. Apache httpd is the general-purpose web server for serving static files and proxying API requests.

Enable Required Modules

macOS: Edit /opt/homebrew/etc/httpd/httpd.conf — uncomment:

LoadModule proxy_module lib/httpd/modules/mod_proxy.so
LoadModule proxy_http_module lib/httpd/modules/mod_proxy_http.so
LoadModule rewrite_module lib/httpd/modules/mod_rewrite.so
LoadModule headers_module lib/httpd/modules/mod_headers.so

Ubuntu:

sudo a2enmod proxy proxy_http rewrite headers

Install Config

macOS:

# Copy the Mindforge Apache config
cp deploy/apache/mindforge.conf /opt/homebrew/etc/httpd/extra/

# Include it in the main httpd.conf (add this line at the end)
echo 'Include /opt/homebrew/etc/httpd/extra/mindforge.conf' >> /opt/homebrew/etc/httpd/httpd.conf

# Restart Apache
brew services restart httpd

Ubuntu:

sudo cp deploy/apache/mindforge.conf /etc/apache2/sites-available/
# Update MINDFORGE_ROOT path in the file
sudo nano /etc/apache2/sites-available/mindforge.conf
sudo a2ensite mindforge
sudo a2dissite 000-default
sudo systemctl restart apache2

8. Access From Any Device

Find your server IP:

# macOS
ipconfig getifaddr en0

# Linux
hostname -I | awk '{print $1}'

Then from any browser on the same network:

:8080
Landing Page
:8001
Student App
:8002
Parent App
:8003
Teacher App
:8004
Admin App

Example: If your server IP is 192.168.1.100, open http://192.168.1.100:8080 to see the landing page.

pm2 restart all                  # Restart all
pm2 stop all                     # Stop all
pm2 monit                        # Real-time dashboard

Apache httpd

# macOS
brew services restart httpd

# Ubuntu
sudo systemctl restart apache2
sudo apache2ctl configtest

PostgreSQL

# macOS
brew services restart postgresql@16
psql -U prayash -l               # List databases

# Ubuntu
sudo systemctl restart postgresql
sudo -u postgres psql -l

11. Troubleshooting

ECONNREFUSED on port 5432

PostgreSQL is not running. Start it:

brew services start postgresql@16    # macOS
sudo systemctl start postgresql      # Ubuntu

Apache — "Address already in use"

lsof -i :8080    # Find process on port
kill <PID>

502 Bad Gateway on API calls

Backend is not running. Check: pm2 status and pm2 logs

Blank page / "Cannot GET /"

Frontend wasn't built. Run npm run build in the frontend directory.

Devices on network can't connect

  1. Check firewall (open ports 8001-8004, 8080)
  2. Verify IP: ipconfig getifaddr en0
  3. Ensure same network/subnet

12. Quick Start (TL;DR)

# 1. Install
brew install node postgresql@16 httpd
npm install -g pm2
brew services start postgresql@16

# 2. Create databases
createdb mindforge && createdb mindforge_admin
createdb mindforge_teacher && createdb mindforge_parent

# 3. Build and deploy
cd /Users/prayash/Project/Mindforge
./scripts/build-and-deploy.sh

# 4. Configure Apache (one-time)
cp deploy/apache/mindforge.conf /opt/homebrew/etc/httpd/extra/
echo 'Include /opt/homebrew/etc/httpd/extra/mindforge.conf' \
  >> /opt/homebrew/etc/httpd/httpd.conf
# Uncomment proxy modules in httpd.conf
brew services restart httpd

# 5. Open http://YOUR_IP:8080
