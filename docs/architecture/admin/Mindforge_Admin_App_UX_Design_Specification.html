1. UX System Overview

Admin app is the control plane for user approvals, fees, payments, and Parent app payment info. It is web-first, desktop-oriented, and optimised for high-safety, high-trust operations.

UX Principles

  • Control-plane clarity – KPIs and lists show current system state at a glance.
  • Safety over speed – confirms high-impact actions like approvals and fee changes.
  • Bulk-friendly – tables and filters support high-volume admin work.
  • Single source of truth – reflect Admin’s authority for approvals and fees.
  • Consistent branding – same Mindforge palette as Teacher/Parent apps.

2. Key Admin Flows

2.1 Review & Approve Users

Sidebar → Users

1. Filter by role (All / Students / Teachers / Parents) and status (Pending / Active / Disabled).
2. See table of users with status badges and Approve / Reject / Activate / Deactivate actions.
3. Open User details panel to view profile and (for parents) linked student information.
4. Approve:
   - Confirmation modal explains default MPIN = 000000 and first-login change requirement.
   - On confirm, status becomes Active and audit log is written.
5. Reject or Disable with confirmation and optional reason.

2.2 Configure Grade Fees & Extra Subject Fees

Sidebar → Fees → Grade fees tab

1. Table shows grade, academic year, total annual fee, last updated, updated by.
2. Click Edit on a grade row:
   - Fee cell becomes editable.
   - Warning banner: "Changes affect fee summaries for all students in this grade."
3. Save and confirm; audit log records change.
4. "Extra subject fees" section links to per-student configuration screen.

2.3 Record & Correct Fee Payments

Sidebar → Payments

1. Search for student, select academic year.
2. See Student fee summary card + Payment history table.
3. Click "Record new payment":
   - Modal with Date, Amount, Mode, Reference.
   - Note: "Admin-only manual entry. Please verify details before saving."
4. Save → payment added to history; summary recalculated.
5. Edit payment:
   - Modal lets admin adjust fields; audit note recommended for future versions.

2.4 Configure Payment Info (Parent App)

Sidebar → Payment Info

1. Card shows QR code preview + text fields for UPI and bank details.
2. Info banner: "This information appears read-only in the Parent app. No payments are processed here."
3. Upload new QR, change UPI/bank/account details.
4. Save changes → last-updated footer refreshed; audit log entry created.
